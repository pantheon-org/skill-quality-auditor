# Plan: Remediate Audit Frontmatter & Validation Gaps

## Goal

Close the validation blind spot around `.context/audits/` files so that:

1. All audit `Analysis.md` and `Remediation.md` files carry valid YAML frontmatter.
2. The pre-commit hooks validate audit files for frontmatter like any other `.context/` file.
3. Index regeneration treats audit files consistently (no silent skip of legacy files).
4. The `hk check` path detects stale indices involving audit file changes.

## Background

Audit files under `.context/audits/<skill>/<date>/` are generated by `reporter/analysis.go` and `reporter/remediation.go`. The frontmatter line was added to both in commit `c1e70ba`, but 5 of 8 existing audit files predate that change and lack frontmatter. Two pre-commit hooks (`context-frontmatter`, `context-index`) explicitly exclude `.context/audits/**`, creating a blind spot. The `regenerate-context-index.sh` script silently skips audit files without frontmatter rather than warning.

## Steps

### 1. Add frontmatter to the 5 stale audit files

Add the standard frontmatter block to each file with `type: audit`, `status: done`, and the appropriate title/date. Pattern:

```yaml
---
title: "Audit: <skill> Analysis — 2026-06-30"
type: audit
status: done
date: 2026-06-30
---
```

### 2. Remove `.context/audits/**` exclusion from `context-frontmatter` hook

In `hk.pkl`, delete the `exclude = List(".context/audits/**")` line from the `context-frontmatter` step (line 32). Audit files now carry frontmatter, so they should be validated like all other `.context/` files.

### 3. Remove `.context/audits/**` exclusion from `context-index` hook

In `hk.pkl`, delete the `exclude = List(".context/audits/**")` line from the `context-index` step (line 37). Changes to audit file frontmatter should trigger index regeneration.

### 4. Update `regenerate-context-index.sh` legacy skip logic

In `.agents/skills/context-index/scripts/regenerate-context-index.sh`, remove the special-case skip on lines 46-47:

```python
if rel.startswith(".context/audits/") and not content.startswith("---\n"):
    continue  # skip legacy audits without frontmatter
```

After all audit files have frontmatter, this code path is dead. Remove it so audit files without frontmatter are flagged as warnings (caught by the `missing` list below), not silently dropped.

### 5. Remove stale-`audits` exemption from `check-context-frontmatter.sh` (if any)

Verify the validation script has no analogous exclusion. If it does, remove it.

### 6. Regenerate `.context/index.yaml`

After all changes, run `regenerate-context-index.sh` to produce a fresh index.

### 7. Run `hk check` to confirm all hooks pass

```bash
hk check
```

## Open Questions

- **Should audit files appear in `.context/index.yaml` under their own `audits` section?** Current decision is no — they're generated artifacts. But once hooks validate them, the index regeneration script will need to handle the `audit` type (it currently only knows `plan`, `finding`, `analysis`, `instruction`). The script currently drops files with unknown types into an `other` bucket — that may suffice, or we add an explicit `audits` section.

- **Should we add a step to regenerate old audit files by re-running `evaluate --store`?** The Go code already generates frontmatter since commit `c1e70ba`. Re-running evaluation on the affected skills would overwrite the stale files with frontmatter-correct versions using the canonical generator, rather than us manually editing them. Downside: it would also overwrite the score data with whatever the current scorer produces (scores may have changed since the original run).

- **Should we fix the `content`-before-assign bug in `regenerate-context-index.sh` (line 46 reads `content` before line 48 assigns it)?** Step 4 removes the buggy skip check entirely, which fixes it implicitly. But if we ever want that skip logic back, the bug must be fixed. Worth calling out in the commit message.
